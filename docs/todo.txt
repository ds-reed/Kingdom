# ======================================================================
#
# Goal: Create a clean, extensible way to handle verb + noun commands
#       so new verbs, nouns, rooms, and custom behaviors can be added
#       without rewriting core parser or game logic.
#
# Key Principles:
#   - Nouns own their specific refusals and behaviors (double dispatch)
#   - Verbs handle general logic, room constraints, and fallback
#   - Parser resolves verb → target noun → verb.execute(state, target)
#   - Refusals: nouns for specifics ("carrot screams"), verbs for generics
#
# High-Level Steps:
#
# 1. Enhance Noun base class with verb interaction hooks
#    - Add: can_handle_verb(verb_name, *args) → (bool, refusal_msg | None)
#    - Add: handle_verb(verb_name, *args) → result_msg | None
#    - Default: allow everything, return None (not handled)
#
# 2. Update Item, Box, Room to override these when needed
#    Examples:
#      - Item.take → check pickupable, return refuse_string
#      - Item.eat (carrot) → return custom scream message
#      - Room.swim → refuse unless in pool room
#      - Room.examine → return rich description if targeted
#
# 3. Modify Verb.execute to support double dispatch
#    Signature: execute(state: GameActionState, target: Noun | None, *args, **kwargs) → str
#    Flow:
#      - If no target: run general action (inventory, score, verbs, etc.)
#      - If target:
#          - Ask target.can_handle_verb(self.verb, *args)
#          - If no → return refusal or generic "Can't {verb} that!"
#          - If yes → call target.handle_verb(self.verb, *args)
#          - If noun returns result → use it
#          - Else fallback to verb's own action(state, target, ...)
#

#    - Split input → verb_token + rest_words
#    - Find Verb by verb_token or synonym
#    - If rest: search current room (items + boxes) for matching noun via matches_reference
#    - Call verb.execute(state, target=noun, *rest_words)
#    - Handle "no target" verbs separately (inventory, look, etc.)

# 5. Update existing actions to new signature
#    - go_action, inventory_action, examine_action → accept target=None
#    - take_action, drop_action, eat_action → accept target: Noun | None
#    - For legacy stubs: keep as-is or migrate to noun.handle_verb when ready
#
# 6. JSON / serialization considerations

#    - e.g., "eat_refusal", "eat_result", or subclass name for special logic
